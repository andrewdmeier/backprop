        newtype ZeroFunc a Source #

        "Zero out" all components of a value. For scalar values, this should just be const 0. For vectors and matrices, this should set all components to zero, the additive identity.

        Each type should ideally only have one ZeroFunc. This coherence constraint is given by the typeclass Zero.

        newtype AddFunc a Source #

        Add together two values of a type. To combine contributions of gradients, so should ideally be information-preserving. For any other valid ZeroFunc, should ideally obey:

        af zf x y -> runAF af x (runZF zf y) == x
           && runAF af (runZF zf x) y == y

        Each type should ideally only have one AddFunc. This coherence constraint is given by the typeclass Add.
